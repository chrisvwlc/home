Views, Controllers and Scope

Controllers are objects which will determine what data gets bound to the view...If the view passes up data, the controller will handle passing that off to a service, etc

the $scope is the GLUE / VIEW MODEL between the view and the controller... The view doesn't have to know about the controller and the controller definitely doesn't want to know about the view...

The ViewModel is the model/data for the view

Angular, when this scope object gets used, will automatically inject a scope object in. This controller will serve as a source of data for the view but the controller shouldn't know about the view - so, how do we do this?

That's what scope is for - the scope will be bound into the view once the view knows about this controller
